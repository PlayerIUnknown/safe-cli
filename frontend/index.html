Install Safe CLI on Endpoint

⚠️ IMPORTANT: Use SOURCE Command

You must use source ./safe-cli-installer.sh (not ./safe-cli-installer.sh) to ensure command aliases are properly activated in your current shell.

One comprehensive installer that handles everything: authentication, registration, and installation type selection:

1

Download Installer

Download the Safe CLI installer script:

2

Make Executable

Make the script executable:

chmod +x safe-cli-installer.sh
3

Source Installer (RECOMMENDED)

IMPORTANT: SOURCE the script to ensure aliases are activated in your current shell:

source ./safe-cli-installer.sh

⚠️ If you run ./safe-cli-installer.sh instead, aliases won't be active in your current shell.

4

Alternative: Execute Script

If you prefer to execute the script (aliases won't be active in current shell):

./safe-cli-installer.sh

⚠️ After execution, you'll need to manually run: source safe-cli-agent.sh to activate aliases.

5

Follow Setup

The installer will ask for:

  • Safe CLI server URL (default: https://your-app.vercel.app)
  • Your username and password
  • Your root user ID
  • Installation type (Global or Single Terminal)

What the Installer Does:

Authentication

Prompts for username/password and authenticates with the dashboard.

Endpoint Registration

Automatically registers the endpoint with the dashboard server.

Installation Type

Lets you choose between global or single-terminal installation.

Error Handling

Includes all fixes for endpoint deletion and connection issues.
